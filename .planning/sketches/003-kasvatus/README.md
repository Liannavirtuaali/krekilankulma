# Sketch 003: Kasvatus-sivu

## Designkysymys
Mikä korttityyli toimii varsarekisterille, jossa on odotetut ja syntyneet varsat?

## Miten avaat
`.planning/sketches/003-kasvatus/index.html` — avaa selaimessa

## Variantit
- **A: Nimi ensin** — varsan nimi isolla, Isä · Emä riville alle, status-badge pienellä
- **B: Vanhemmat ensin** — Isä × Emä pääotsikkona, varsan nimi kursiivina alle
- **C: Kompakti lista** — pienikokoinen kuva, nimi + vanhemmat + vuosi samalla rivillä, status-badge oikeassa reunassa, suodatuspainikkeet

## Mitä katsoa
- Kumpi tietohierarkia toimii paremmin — nimi vai vanhemmat ensin?
- Miten "Nimetön varsa" (odotettu) näyttää eri varianteissa?
- Onko C:n kompaktisuus hyöty (enemmän varsoja kerralla) vai haittaako luettavuutta?
- Toimiiko suodatus (C) — onko se tarpeellinen vai lisäkompleksisuus?
