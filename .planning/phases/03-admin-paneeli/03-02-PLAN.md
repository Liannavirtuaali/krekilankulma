<task type="auto">
  <name>Task 2: Hevosen lisäyslomake (horse_add.php)</name>
  <files>public/admin/horse_add.php</files>
  <action>
Luo `public/admin/horse_add.php`.

1. Require `../src/includes/db.php`, kutsu `requireLogin()`.
2. CSRF-token sessioon jos ei ole.
3. Hae dropdowneja varten:
   - `SELECT id, name FROM disciplines ORDER BY name`
   - `SELECT id, name FROM levels ORDER BY name`
   - `SELECT id, name FROM horses WHERE is_deleted = 0 ORDER BY name` — sire_id ja dam_id dropdowneihin (sama lista molemmille)
4. POST-käsittely:
   - Tarkista CSRF.
   - Sanitoi kaikki kentät sanitize():llä.
   - Pakolliset kentät: `name` — jos tyhjä, aseta `$errors[]`.
   - Generoi slug: `$slug = slugify($name)`. Tarkista onko slug jo käytössä, jos on lisää luku loppuun (`$slug . '-2'` jne.).
   - INSERT horses kaikki kentät prepared statementilla.
   - Onnistuessa ohjaa `horses.php?added=1`.
5. Lomakekentät (kaikki horses-taulun kentät):
   - `name` (text, pakollinen), `call_name` (text), `vh_id` (text)
   - `breed` (text), `birth_date` (date), `gender` (select: ori/tamma/ruuna/tuntematon)
   - `color` (text), `height_cm` (number, min=100 max=200)
   - `discipline_id` (select, haettu DB:stä), `level_id` (select, haettu DB:stä)
   - `owner_name`, `owner_email`, `breeder_name`, `breeder_email`
   - `importer_name`, `importer_email`
   - `sire_id` (select — Isä, tyhjä vaihtoehto "— ei valittu —"), `dam_id` (select — Emä)
   - `evm` (text — VH-rekisterinumero tai vastaava), `profile_url` (url)
   - `description` (textarea), `pedigree_notes` (textarea)
6. Näytä `$errors` listana jos on.
7. Sisällytä admin_header.php (`$pageTitle = 'Lisää hevonen'`) ja admin_footer.php.
  </action>
  <verify>
    <automated>php -l public/admin/horse_add.php</automated>
  </verify>
  <done>horse_add.php näyttää lomakkeen kaikilla kentillä. POST luo uuden hevosen DB:hen oikealla slugilla ja ohjaa horses.php:lle.</done>
</task>

<task type="auto">
  <name>Task 3: Muokkauslomake ja pehmeä poisto (horse_edit.php, horse_delete.php)</name>
  <files>public/admin/horse_edit.php, public/admin/horse_delete.php</files>
  <action>
**horse_edit.php:**
1. Require `../src/includes/db.php`, kutsu `requireLogin()`.
2. Lue `$id = (int)($_GET['id'] ?? 0)` — jos 0 tai hevosta ei löydy, ohjaa `horses.php`.
3. Hae hevonen: `SELECT * FROM horses WHERE id = :id AND is_deleted = 0`.
4. Hae dropdownit samoin kuin horse_add.php:ssä.
5. POST-käsittely:
   - Tarkista CSRF.
   - Sanitoi kentät, validoi pakollinen `name`.
   - Jos nimi muuttui, generoi uusi slug (sama duplicate-check kuin add:ssä, mutta sulje nykyinen id pois: `WHERE slug = :slug AND id != :id`).
   - UPDATE horses SET ... WHERE id = :id.
   - Onnistuessa ohjaa `horses.php?updated=1`.
6. Lomake on identtinen horse_add.php:n kanssa, mutta kenttien `value`-attribuutit esitäytetty DB:stä haetulla datalla. CSRF-token piilotettu kenttä. Myös `id` piilokentässä.
7. `$pageTitle = 'Muokkaa: ' . e($horse['name'])`.

**horse_delete.php:**
1. Require `../src/includes/db.php`, kutsu `requireLogin()`.
2. Hyväksy VAIN POST. Jos GET, ohjaa `horses.php`.
3. Tarkista CSRF.
4. `$id = (int)($_POST['id'] ?? 0)`.
5. Pehmeä poisto: `UPDATE horses SET is_deleted = 1, deleted_at = NOW() WHERE id = :id AND is_deleted = 0`.
6. Ohjaa `horses.php?deleted=1`.

TÄRKEÄÄ: Älä käytä DELETE-kyselyä. Käytä aina `UPDATE ... SET is_deleted = 1` (ADMIN-03).
  </action>
  <verify>
    <automated>php -l public/admin/horse_edit.php && php -l public/admin/horse_delete.php</automated>
  </verify>
  <done>horse_edit.php lataa hevosen tiedot ja tallentaa muutokset. horse_delete.php asettaa is_deleted=1 (ei tuhoa riviä). Molemmat vaativat kirjautumisen ja CSRF-tokenin.</done>
</task>
