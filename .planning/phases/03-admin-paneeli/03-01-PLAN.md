<task type="auto">
  <name>Task 1: Kirjautumislomake ja POST-käsittely (login.php)</name>
  <files>public/admin/login.php</files>
  <action>
Luo tiedosto `public/admin/login.php`. Tämä on AINOA admin-sivu joka ei kutsu requireLogin().

Logiikka:
1. Require `../src/includes/db.php` (käynnistää session automaattisesti getDB():n kautta).
2. Jos käyttäjä on jo kirjautunut (isLoggedIn()), ohjaa `index.php`:lle.
3. CSRF-token: jos ei ole sessiossa, generoi `$_SESSION['csrf_token'] = bin2hex(random_bytes(32))`.
4. POST-käsittely:
   - Tarkista CSRF: `hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'] ?? '')` — jos epäonnistuu, näytä virhe "Virheellinen pyyntö."
   - Sanitoi syöte: `$username = sanitize($_POST['username'] ?? '')`.
   - Hae käyttäjä: `SELECT id, username, password FROM admin_users WHERE username = :username LIMIT 1`.
   - Tarkista salasana: `password_verify($password, $row['password'])`.
   - Onnistuessa: `session_regenerate_id(true)`, aseta `$_SESSION['admin_logged_in'] = true`, `$_SESSION['admin_id'] = $row['id']` ja `$_SESSION['admin_username'] = $row['username']`, ohjaa `index.php`:lle.
   - Epäonnistuessa: aseta `$error = 'Väärä käyttäjätunnus tai salasana.'` (älä paljasta kumpi on väärin).
5. HTML-lomake: `method="post"`, `action=""`, kentät `username` ja `password`, piilotettu kenttä `csrf_token` (arvo sessiosta), nappi "Kirjaudu sisään".
6. Näytä `$error` jos asetettu, e()-suojattuna.
7. Sivun rakenne: oma minimaalinen HTML (ei admin_header.php — käyttäjä ei ole vielä kirjautunut). Käytä inline CSS tai linkki `../assets/css/style.css`.
  </action>
  <verify>
    <automated>php -l public/admin/login.php</automated>
  </verify>
  <done>login.php latautuu virheettä. Oikea salasana kirjaa sisään ja ohjaa index.php:lle. Väärä salasana näyttää virheen. CSRF-token on lomakkeessa.</done>
</task>

<task type="auto">
  <name>Task 2: Uloskirjautuminen (logout.php) ja admin-etusivu (index.php)</name>
  <files>public/admin/logout.php, public/admin/index.php</files>
  <action>
**logout.php:**
1. Require `../src/includes/db.php`.
2. Käytä requireLogin() (estää suoran pääsyn).
3. CSRF-tarkistus POST:lle (uloskirjautumisnappi on POST-lomake).
4. `$_SESSION = []`, `session_destroy()`.
5. Ohjaa `login.php`:lle.

Vaihtoehto: logout voidaan tehdä myös GET:llä token-parametrilla (`?token=...`). Valitse POST-lomake — turvallisempi CSRF:n kannalta.

**index.php (admin-etusivu / dashboard):**
1. Require `../src/includes/db.php`.
2. Kutsu `requireLogin()` heti ensimmäisenä.
3. Hae tilastot dashboardia varten:
   - Hevosten määrä: `SELECT COUNT(*) FROM horses WHERE is_deleted = 0`
   - Kuvien määrä: `SELECT COUNT(*) FROM horse_photos`
   - Kilpailujen määrä: `SELECT COUNT(*) FROM competitions`
4. Sisällytä `includes/admin_header.php` ja `includes/admin_footer.php`.
5. Näytä dashboard: tervetuloa-teksti (`Tervetuloa, <?= e($_SESSION['admin_username']) ?>!`), pikavalinnat linkeillä (Hevoset, Lisää hevonen), tilastokortit (Hevosia X kpl, Kuvia Y kpl, Kilpailuja Z kpl).
6. Uloskirjautumisnappula on POST-lomake admin_footer.php:ssä tai navigaatiossa.
  </action>

  </verify>
  <done>logout.php tuhoaa session ja ohjaa login.php:lle. index.php vaatii kirjautumisen ja näyttää dashboard-näkymän tilastoilla.</done>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| selain → login.php | Käyttäjäsyöte (username, password) — täysin epäluotettu |
| sessio → admin-sivut | PHP-session validointi — luotettu jos session_regenerate_id() on kutsuttu |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | login.php / admin_users | mitigate | password_verify() + bcrypt, ei MD5/SHA1 |
| T-03-02 | Tampering | login.php POST | mitigate | CSRF-token: bin2hex(random_bytes(32)) + hash_equals() |
| T-03-03 | Repudiation | session | mitigate | session_regenerate_id(true) onnistuneen kirjautumisen jälkeen (session fixation -esto) |
| T-03-04 | Information Disclosure | login virheviesti | mitigate | Geneerinen virheviesti — ei paljasteta kumpi (username/password) on väärin |
| T-03-05 | Elevation of Privilege | admin-sivut | mitigate | requireLogin() jokaisen admin-sivun ensimmäinen kutsu |
| T-03-06 | Tampering | logout | mitigate | POST-lomake CSRF-tokenilla, ei GET-linkki |
</threat_model>
