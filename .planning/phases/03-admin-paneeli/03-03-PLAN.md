<task type="auto">
  <name>Task 1: Kuvien listaus ja latauslomake (photos.php)</name>
  <files>public/admin/photos.php</files>
  <action>
Luo `public/admin/photos.php`.

1. Require `../src/includes/db.php`, kutsu `requireLogin()`.
2. `$horse_id = (int)($_GET['horse_id'] ?? 0)` — jos 0 tai hevosta ei löydy, ohjaa `horses.php`.
3. Hae hevosen perustiedot: `SELECT id, name FROM horses WHERE id = :id AND is_deleted = 0`.
4. Hae nykyiset kuvat: `SELECT id, filename, original_name, sort_order FROM horse_photos WHERE horse_id = :horse_id ORDER BY sort_order ASC`.
5. CSRF-token sessioon jos ei ole.

**POST-käsittely (tiedoston lataus):**
a) Tarkista CSRF: `hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'] ?? '')`.
b) Tarkista kuvamäärä: `SELECT COUNT(*) FROM horse_photos WHERE horse_id = :horse_id`. Jos >= MAX_PHOTOS_PER_HORSE (5), näytä virhe "Hevosella on jo 5 kuvaa. Poista ensin vanha kuva." — ÄLÄ jatka.
c) Tarkista että tiedosto on ladattu: `$_FILES['photo']['error'] === UPLOAD_ERR_OK`.
d) Tarkista tiedostokoko: `$_FILES['photo']['size'] <= MAX_UPLOAD_SIZE` (5MB). Virhe jos ylittyy.
e) MIME-tarkistus finfo_file():llä:
   ```php
   $finfo = new finfo(FILEINFO_MIME_TYPE);
   $mime = $finfo->file($_FILES['photo']['tmp_name']);
   if (!in_array($mime, ALLOWED_MIME_TYPES, true)) {
       $error = 'Sallitut tiedostotyypit: JPEG, PNG, GIF, WebP.';
   }
   ```
f) Tiedostopäätteen tarkistus:
   ```php
   $ext = strtolower(pathinfo($_FILES['photo']['name'], PATHINFO_EXTENSION));
   if (!in_array($ext, ALLOWED_EXTENSIONS, true)) {
       $error = 'Sallitut päätteet: jpg, jpeg, png, gif, webp.';
   }
   ```
g) Generoi uniikki tiedostonimi: `$filename = uniqid('img_', true) . '.' . $ext`. ÄLÄ käytä alkuperäistä nimeä polkuna.
h) Siirrä tiedosto: `move_uploaded_file($_FILES['photo']['tmp_name'], UPLOADS_DIR . $filename)`.
i) Hae seuraava sort_order: `SELECT MAX(sort_order) FROM horse_photos WHERE horse_id = :horse_id`. Lisää 1.
j) INSERT horse_photos: `(horse_id, filename, original_name, sort_order)`.
k) Onnistuessa ohjaa `photos.php?horse_id={id}&uploaded=1`.

**HTML-näkymä:**
- Otsikko: "Kuvat — [hevosen nimi]"
- Nykyiset kuvat: thumbnail-lista (kuva, alkuperäinen nimi, poisto-nappi POST-lomakkeena photo_delete.php:lle).
- Thumbnail-src: `UPLOADS_URL . e($photo['filename'])`.
- Latauslomake (vain jos kuvamäärä < 5): `<form method="post" enctype="multipart/form-data">`, CSRF-token, `<input type="file" name="photo" accept="image/*">`, nappi "Lataa kuva".
- Jos kuvamäärä = 5: näytä viesti "Hevosella on jo 5 kuvaa." latauslomakkeen sijaan.
- Linkki takaisin: `horses.php`.
- Sisällytä admin_header.php ja admin_footer.php.

<task type="auto">
  <name>Task 2: Kuvan poisto (photo_delete.php)</name>
  <files>public/admin/photo_delete.php</files>
  <action>
Luo `public/admin/photo_delete.php`.

1. Require `../src/includes/db.php`, kutsu `requireLogin()`.
2. Hyväksy VAIN POST. Jos GET, ohjaa `horses.php`.
3. Tarkista CSRF: `hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'] ?? '')`.
4. `$photo_id = (int)($_POST['photo_id'] ?? 0)`.
5. `$horse_id = (int)($_POST['horse_id'] ?? 0)` — tarvitaan uudelleenohjaukseen.
6. Hae kuvan tiedot: `SELECT id, filename, horse_id FROM horse_photos WHERE id = :photo_id`.
7. Jos ei löydy, ohjaa `horses.php`.
8. Tarkista omistajuus: varmista että `$row['horse_id'] === $horse_id` (estää muiden hevosten kuvien poiston manipuloidulla POST:lla).
9. Poista tiedosto: `$filepath = UPLOADS_DIR . $row['filename']`. Jos tiedosto on olemassa (`file_exists()`), kutsu `unlink($filepath)`.
10. Poista DB-rivi: `DELETE FROM horse_photos WHERE id = :photo_id`.
11. Ohjaa `photos.php?horse_id={horse_id}&deleted=1`.

HUOM: Tässä on poikkeus is_deleted-periaatteesta — kuvat poistetaan oikeasti sekä tiedostojärjestelmästä että DB:stä (ei softdelete), koska tiedostot vievät tilaa.
