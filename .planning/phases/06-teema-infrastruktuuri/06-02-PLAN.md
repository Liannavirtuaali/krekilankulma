Output: `public/src/includes/theme.php` (uusi), `public/pages/index.php` (yksi rivi lisätty).

<threat_model>
- **Path traversal / LFI (V5 Input Validation, STRIDE Tampering + Info Disclosure):** `resolveThemePath($subPath)` saa Phase 8:ssa polkukomponentteja; pahantahtoinen syöte kuten `../../etc/passwd` tai `../../../config.php` yrittäisi vuotaa palvelimen tiedostoja. Mitigation: `realpath()` normalisoi polun (resoloi `../`, symlinkit, dekoodatut merkit) ja `str_starts_with($real, THEME_PATH)` -prefix-check (trailing DIRECTORY_SEPARATOR) varmistaa että lopullinen polku pysyy teemakansion sisällä. Sekä aktiivinen että default-fallback tekevät saman tarkistuksen. Verifioitu Task 2:n behavior-testillä (`../../etc/passwd` → false).
- **Theme name injection (V5 Input Validation, STRIDE Tampering):** DB:stä luettu `active_theme`-arvo voisi teoriassa sisältää polkutraversaalimerkkejä. Mitigation: `preg_match('/^[a-zA-Z0-9_-]+$/')` allowlist ENNEN mitään tiedosto-operaatiota; epäkelpo nimi korvataan `'default'`:lla. Null-byte injection (`theme\0name`) torjutaan samalla allowlistilla (null-tavu ei kuulu merkistöön).
- **Admin-eristyksen rikkominen (V4 Access Control):** Jos joku lisää `require_once theme.php`:n db.php:hen, admin-sivut alkavat ladata shimin ja tehdä turhan DB-kyselyn jokaisella admin-pyynnöllä (D-09 rikko). Mitigation: theme.php:n header-docblock sisältää eksplisiittisen varoituksen; theme.php require_once-kutsutaan VAIN public/pages/index.php:stä, ei koskaan db.php:stä tai helpers.php:stä. Verifioitu: theme.php ei esiinny db.php:n require-ketjussa.
- **SQL injection active_theme-luvussa (V5):** Mitigation: prepared statement named placeholderilla (`:k`), ei merkkijonokonkatenaatiota — noudattaa olemassa olevaa SEC-01-patternia.
</threat_model>
